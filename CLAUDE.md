### 선택 개선 (사용자 판단 필요)

- **accent 대비 미달 2건.** 원쁠원 `#2E8B47`(3.99:1), 가마솥 `#B06A28`(3.96:1)이 WCAG AA(4.5:1) 미달.
  기존 페이지의 정체성 색이라 유지했다. 본문 텍스트엔 안 쓰이고 라벨·아이브로우에만 쓰인다.
  고치려면 각 페이지 `:root`의 `--accent` 한 줄. (여우숲 5.70 / 엔드필드 5.31 / 헤스티 4.45는 통과)
- **원쁠원 판독 폭 880px** — 나머지 4개는 1120px. 완성된 페이지의 레이아웃 결정이라 건드리지 않았다.
  통일하려면 `--maxw`와 `.inner`/`.row`/`.hero-inner`의 `max-width`를 함께 고쳐야 한다(테이블·갤러리 재검증 필요).
- 원쁠원에 `<meta name="description">`이 없다. 5개 중 유일. SEO·공유 시 아쉬움.

## 2. 이번에 내린 판단과 근거 (되돌리기 전에 읽을 것)

### 2-1. "최신 세대"의 정의

토큰 스키마로 세대가 갈렸다. 측정 결과:

- **최신 = 역기획 3종**(헤스티·엔드필드·가마솥). `--font-title/body/mono`, `--slate`, `--observed`, `--patch-soft`, `--maxw` 스키마가 완전 동일. 색 역할까지 주석으로 명세.
- **중간 = 원쁠원.** 팔레트 값은 최신과 동일한데 변수명만 구식(`--disp/--body/--mono`).
- **구세대 = 여우숲.** 다크 테마(#080b11) + Noto 폰트. Pretendard·Hahmlet 미사용.

→ 여우숲은 **가장 오래된** 페이지였다. 최신 3종의 시스템으로 끌어올렸다.

### 2-2. 여우숲 — 히어로의 어둠은 살렸다

최신 세대는 **어두운 히어로(slate) → 밝은 본문 → 어두운 푸터** 패턴이다.
여우숲의 다크 히어로는 이 패턴과 충돌하지 않고 오히려 부합하므로, 히어로의 밤 연출·빛줄기는 유지하고 본문만 라이트로 전환했다.
"통일"은 다크를 전부 지우는 게 아니라 같은 시스템 문법을 쓰는 것이다.

### 2-3. 여우숲 accent = `#4A6B2A` 모스 그린

원래 앰버 `#F2B450`은 라이트 배경에서 **대비 1.71:1로 사용 불가**. 대비를 확보한 앰버(`#96661C`)는 가마솥 앰버와 **색상환 7° 차이**로 충돌.
색상환 빈 구역은 황록(60~110°)과 보라(240~280°)뿐이고 빨강·파랑은 시스템 예약색(patch/link)이다.
→ 모스 그린 선택. 게임 이름 그대로 '숲'에 묶이고, 원쁠원 켈리그린과 46° 이격(기존 가마솥↔헤스티는 2° 차이로 이미 붙어 있음), 대비 5.70:1.

### 2-4. 레거시 별칭을 남긴 이유

여우숲·원쁠원은 구세대 변수명이 **본문 인라인 `style`**에 박혀 있었다(여우숲 15곳). 본문을 건드리지 않기로 했으므로 `:root`에서 별칭으로 연결했다.
`--amber2:var(--accent)` / `--teal:var(--observed)` / `--mono:var(--font-mono)` 등.
**이 줄을 지우면 해당 요소 색이 깨진다.** 자세한 내용은 README의 "레거시 별칭".

### 2-5. 셸은 페이지 토큰을 상속한다

`assets/style.css`의 셸은 색·폭을 하드코딩하지 않고 `var(--accent)`, `var(--maxw)`를 쓴다.
→ 같은 마크업이 페이지마다 그 페이지의 색과 폭에 자동 정렬된다. 원쁠원(880px)과 나머지(1120px)가 각각 맞는 이유.
**셸에 색을 하드코딩하지 말 것.** 이 성질이 깨진다.

## 4. 원칙 (유지)

- 일관성 > 화려함. 공통 요소는 `assets/style.css` 단일 소스로만 통제.
- 정보 위계 = 스캔 → 깊이 (Input→Condition→Output 철학의 시연).
- 실제 콘텐츠는 추측하지 않고 사용자에게 요청.
- 프로젝트 페이지 본문은 **의미 변경 금지**. 구조적 주입(셸)과 기계적 치환(경로·토큰명)만 허용.
