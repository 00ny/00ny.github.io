# 디자인 리뉴얼 브리프 — Cinematic Dark (Claude Code 작업 지시서)

> **용도:** 배포 중인 포트폴리오(`https://00ny.github.io`)의 디자인 리뉴얼 작업 지시서.
> **사용법:** 이 파일을 저장소 루트에 `DESIGN_RENEWAL.md`로 저장한 뒤, Claude Code에 *"DESIGN_RENEWAL.md대로 리뉴얼해줘"* 라고 지시. 기존 `CLAUDE.md`(구조·콘텐츠 원칙)는 그대로 유효하며, 이 문서는 **비주얼 레이어만** 덮어쓴다.

---

## 0. 배경 — 왜 바꾸는가

사용자 진단: 현재 디자인이 (1) 단조롭고 올드하다 (2) 첫 화면 몰입이 안 된다(상하단 흰색) (3) 프리미엄 무드가 없다.
근본 원인은 **대비(Contrast)의 부재** — 밝은 단색 배경 + 균일한 카드 그리드 + 균일한 타입 스케일.

**한 줄 방향: "문서를 무대(Stage)로 바꾼다."**
게임 포트폴리오는 콘텐츠(키아트·스크린샷)가 발광체다. 배경은 어둡게 물러나고 콘텐츠가 빛난다.

## 1. 작업 범위 — 엄격히 한정

**할 것:**
- `assets/style.css` — 디자인 토큰 전면 교체 + 컴포넌트 스타일 조정 (단일 소스 원칙 유지)
- `index.html` — 히어로 풀 블리드 구조, 카드 위계, 넘버링을 위한 **최소한의 마크업 조정**

**하지 말 것:**
- 정보 구조·콘텐츠·문안 변경 금지 (히어로 헤드라인 1건 제외, §4 참고)
- 프로젝트 페이지(2depth) 리디자인 금지 — §7의 캐스케이드 검수만
- 프레임워크·빌드 도구 도입 금지. 순수 HTML/CSS 유지, JS는 기존 수준 최소 유지

## 2. 디자인 컨셉

- **키워드:** `Cinematic Dark` — 스튜디오 쇼케이스 무드. "개인 블로그"가 아니라 "게임 스튜디오의 프로젝트 페이지"처럼.
- **금지:** 네온 다색, 과한 글로우, 장식 그래픽, 요란한 애니메이션. 프리미엄 = 덜어내고 대비를 키우는 것.

## 3. 디자인 토큰 (style.css `:root` 에 정의)

### 3-1. 컬러 — 60/30/10
```css
:root {
  --bg:        #0E0F13;                  /* 베이스. 순흑 금지 */
  --surface:   #16181E;                  /* 카드·패널 */
  --surface-2: #0B0C0F;                  /* 푸터 등 한 단계 더 어두운 마감 */
  --border:    rgba(255,255,255,.08);    /* 옅은 층위 보더 */
  --text:      rgba(255,255,255,.87);    /* 본문 */
  --text-sub:  rgba(255,255,255,.60);    /* 보조 */
  --accent:    #E98A3C;                  /* 엠버(노을). 시작값 — 아래 참고 */
  --accent-strong: #F5A45C;              /* 호버/강조 */
}
```
- **엠버 미세 조정:** 최종 값은 원쁠원(노을 도시)·여우숲 키아트의 노을 톤과 나란히 놓고 어울리게 조정 가능. 단 **채도 과다(형광) 금지.**
- **액센트 사용처 제한(10% 미만):** 히어로 카피 일부 강조, 링크/호버, 카테고리 칩, 넘버링. **한 화면에서 액센트가 세 곳 이상 동시에 눈에 띄면 초과.**
- **텍스트 대비:** 본문 영역은 WCAG AA 대비비 충족을 기준으로 검수.

### 3-2. 타이포 — 3단 대비
- **Display:** 히어로 헤드라인. `clamp()`로 화면폭 반응(모바일에서도 크게, 데스크톱에서 압도적으로). 굵기 700+, 행간 타이트.
- **Body:** Pretendard 유지, 16~17px, 행간 1.7 내외.
- **Micro label:** 섹션 아이브로우(Resume/Projects/Elsewhere) — 영문 대문자, letter-spacing 넓게, `--text-sub` 톤. 기존 장치 유지·강화.

### 3-3. 모션 — 딱 두 종류
- 히어로 헤드라인 로드 시 페이드업 1회 (600ms 내, 1회성).
- 카드 호버: 이미지 미세 줌(1.03 내외) + 텍스트 부상, 200ms대 ease.
- 그 외 스크롤 애니메이션·패럴랙스 금지.

## 4. 히어로(첫 화면) 명세 — 최우선

1. **풀 블리드:** 히어로 높이 `100vh`(모바일 `100svh`). 상하단 흰 띠 제거.
2. **헤더 투명화:** 헤더를 히어로 위에 겹침(absolute/fixed). 스크롤 시 `--bg` 배경으로 전환(블러 선택).
3. **배경:** 대표 키아트(또는 기존 캐러셀)를 풀 블리드로 깔고 **다크 스크림**(하단으로 갈수록 `--bg`에 수렴하는 그라디언트) 오버레이 → 다음 섹션과 끊김 없이 연결.
4. **헤드라인 교체 — 유일하게 허용된 문안 변경:**
   - 현재 "배현이라는 게임 기획자 · 신입의 포트폴리오"는 헤더와 중복되는 명패. **관점을 말하는 한 문장**으로 교체.
   - 마크업엔 `{{HERO_HEADLINE}}` 플레이스홀더로 넣고, **사용자에게 문안을 요청**할 것. 방향 참고: "플레이어가 정보를 읽고, 리스크를 계산하고, 전략을 만들게 한다"는 설계 철학의 선언형 문장. 이름·직군("배현 · 게임 기획")은 보조 행으로.
5. **스크롤 유도:** 하단 중앙에 미세한 스크롤 인디케이터 1개.

## 5. 인덱스 섹션 명세

### 5-1. 프로젝트 카드
- **비대칭 위계:** 대표작 1편을 와이드 피처드 카드로, 나머지 4편은 서브 그리드. **어느 프로젝트를 피처드로 할지 사용자에게 확인**(임시로 여우숲).
- **에디토리얼 넘버링:** 각 카드에 `01`–`05` 인덱스(micro label 스타일, 액센트 or sub 톤).
- 카드 배경 `--surface` + `--border`, 모서리 절제(과한 라운드 금지).

### 5-2. 이력 섹션
- 표 형식 유지. 행 구분은 보더 최소화 + 여백으로. 긴 텍스트 가독 우선(`--text` 대비 확보).

### 5-3. 푸터
- `--surface-2`로 마감. 흰색 완전 제거. "막이 내려가는" 종결감.

## 6. 이미지 트리트먼트 — 톤 통일

- 5개 프로젝트 비주얼 톤이 제각각(노을 도시 / 밤 숲 / 사막 / 수몰 목조 / 캐릭터 아트).
- **모든 썸네일에 공통 규칙 적용:** 하단 다크 그라디언트 스크림 + 동일한 채도/명도 보정(CSS filter 수준). 원본 이미지 파일은 수정하지 않고 CSS로 처리.

## 7. 캐스케이드 처리 원칙 (2depth)

- `style.css`는 전 페이지 공유이므로 토큰 교체 시 **프로젝트 페이지도 자동으로 다크 전환됨.**
- 이번 범위: 2depth는 **가독성 검수만** — 긴 명세 본문·표·`<details>`가 다크에서 읽히는지 확인하고, 깨지는 부분만 최소 보정.
- 리디자인 욕심 금지. 2depth 개선 필요사항은 **목록으로 사용자에게 보고**만.

## 9. 사용자에게 물어볼 것 (작업 중 확인 필수)

1. 히어로 헤드라인 최종 문안 (`{{HERO_HEADLINE}}`)
2. 피처드 카드로 세울 대표작 1편
3. 엠버 최종 톤 (시작값 대비 더 붉게/노랗게 여부 — 스크린샷 비교로 제시)

## 10. 완료 기준

- 첫 화면이 풀 블리드 다크 히어로로 뜨고, 상하단 흰 영역이 완전히 사라짐
- 스크롤 전체에서 배경 단절(흰 띠) 0회
- 카드에 위계(피처드 1 + 서브 4)와 넘버링이 보임
- 토큰이 `:root`에 집중되어 이후 색·타입 수정이 style.css 한 곳에서 가능
- 커밋 후 push → `https://00ny.github.io` 반영 확인

```
git add -A
git commit -m "design: Cinematic Dark 리뉴얼 (P0-P1)"
git push origin main
```
