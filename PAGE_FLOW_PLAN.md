# 페이지 흐름 재배치 기획서 (IA v2) — 이력서 아래 영역의 순서 재도출

> **상태: 기획만 완료 (2026-07-19). 구현 전 — 사용자 확정 대기.**
> 발단: 사용자 질문 2개. ① "5편은 캐러셀이 소개하니 프로젝트 카드를 맨 아래로?" ② "스킬툴이 긴장 곡선·게임 분석보다 위인 게 옳은가?"
> 결론 요약: **①은 기각(오히려 위로 당긴다), ②는 지적이 옳다(레벨바만 부록층으로 내린다).**

---

## 1. 진단 — 현행 순서의 구조적 문제 3가지

현행: `히어로 → 이력서(스킬 포함) → 분석·리뷰 → 기획서 5편(+긴장 곡선) → 언리얼 → 다른 기록 → 지금 하는 게임`

실측(뷰포트 800px 기준, 전체 13,232px):

| 지점 | 위치 | 비율 |
|---|---|---|
| 이력서 시작 | 600px | 5% |
| **스킬 패널(레벨바 22종)** | 3,179px~4,362px (높이 1,183px) | 24~33% |
| 분석·리뷰 | 4,434px | 34% |
| **첫 기획서 카드** | 6,188px | **47%** |
| 긴장 곡선 | 7,756px | 59% |
| 언리얼 | 8,364px | 63% |

**문제 1 — 가치 경사 역전 (Q2의 답: 옳지 않다).**
채용 스캔은 위→아래로 단조 감쇠한다. 그런데 현행은 **범용 정보(자기평가 레벨바 22종)가 차별화 증거(긴장 곡선·리뷰·기획서)보다 위**에 있다.
레벨바는 모든 지원자가 갖고 있는 형식이고, 긴장 곡선("내 좋아하는 레벨에 '실패'라 적는다")은 이 포트폴리오에만 있는 형식이다.
주의 예산이 가장 비싼 구간(상단 1/3)을 가장 싼 정보가 점유하고 있다.

**문제 2 — 코어 도달 깊이.** 이 사이트의 상품은 기획서 5편인데, 카드(실제 소구 문안)까지 전체의 47%를 스크롤해야 한다.
이탈이 일어나는 채용 스캔에서 절반은 코어를 못 보고 떠난다는 뜻.

**문제 3 — "기획서 밖" 카피의 시점 어긋남.** 분석·리뷰 섹션의 lede는 "게임을 뜯어보는 일은 **문서 밖에서도** 이어집니다"인데,
독자는 아직 그 '문서(기획서 5편)'를 만나지 않았다. 이 섹션이 7건 개편 때 "이력서 바로 다음"으로 배치된 건
당시엔 **유일한 추가 증거**였기 때문 — 지금은 긴장 곡선·언리얼·플레이 기록까지 부록층이 생겨 전제가 바뀌었다.

## 2. 질문 2개에 대한 판단

### Q1. "기획서 5편을 맨 아래로?" → **기각. 반대로 이력서 바로 다음으로 당긴다.**

전제("캐러셀이 이미 소개한다")는 절반만 맞다:

| | 캐러셀 | 카드 그리드 |
|---|---|---|
| 보여주는 것 | 제목+한 줄 태그라인, **한 번에 1장** (조작해야 다음) | 5편 전체를 **한눈에 스캔** + 컨셉 문안·분량·태그 |
| 역할 | 티저(관심 유발) | **소구층(선택 근거 제공)** |

즉 캐러셀은 그리드의 대체재가 아니라 예고편이다. 코어 증거를 맨 아래로 보내면 ① 스크롤 감쇠로 도달률이 깎이고,
② 맨 아래는 이미 #play가 의도적으로 잡은 자리(인간적 마무리 — 사용자 확정)라 피날레가 충돌하며,
③ 긴장 곡선 부록(5편 평가)·언리얼("직접 세운 기록 ↓" 브릿지)의 인접 서사가 전부 꼬인다.
사용자가 느낀 진짜 문제는 "프로젝트가 두 번 나온다"가 아니라 **"코어까지 너무 멀다"**다. 처방은 반대 방향 — 앞으로 당기기.

### Q2. "스킬툴이 긴장 곡선·게임 분석보다 위인 게 옳은가?" → **옳지 않다. 단, 전부 내리진 않는다.**

스킬 패널은 두 층위가 섞여 있다:
- **게임 엔진 검증 배지 3종** — "자기평가 대신 결과물로"라는 이 사이트만의 차별점. **이력서에 잔류**(위치 가치 있음).
- **레벨바 22종** — 범용 정보. **부록층(분석·리뷰 뒤)으로 이동.** 이동하면서 lede 한 줄로 정직성 프레임을 붙인다:
  "숙련도는 자기평가입니다 — 실제 증명은 위의 결과물들입니다." (범용 정보를 겸손 카피로 감싸 브랜드에 편입)

## 3. 확정안 — IA v2

```
1. 캐러셀 히어로            (티저 · 헤드라인)
2. #resume  이력 · 활동      (벤토 2행 + 슬림 '게임 엔진 검증' 행 — 레벨바 제거로 ~1,100px 단축)
3. #projects 기획서 5편      (코어 — 이력서 직후로 승격) ← Q1 처방
   └ #tension 긴장 곡선 부록  (5편 평가 — 인접 유지)
4. #unreal  세 번째 엔진      (설계→구현 — tension 브릿지 인접 유지)
5. #analysis 기획서 밖의 게임 분석 (리뷰·유튜브 — "기획서 밖" 카피가 이제 문자 그대로 성립) ← 문제 3 해소
6. #skills  기술 · 도구       (레벨바 22종 — 신설 위치, 목차 미포함) ← Q2 처방
7. #links   다른 기록         (외부로 나가는 출구)
8. #play    지금 하는 게임    (인간적 마무리 — 불변)
```

**서사 검증:** 헤드라인 "뜯어보고 → 만듭니다"의 '뜯어보기' 1차 증명은 리뷰가 아니라 **역기획서 2편**(5편 안)이다.
따라서 5편이 앞으로 와도 헤드라인과의 호응은 유지되고, 리뷰·유튜브는 "문서 밖에서도 계속된다"는 증폭재로서 뒤가 더 자연스럽다.
가치 경사: 코어(5편) → 평가(긴장) → 엔진(언리얼) → 외부 활동(리뷰) → 도구(레벨바) → 링크 → 사람. 단조 하강 ✓.

**효과 추정:** 첫 기획서 카드 도달 47% → **약 20% 지점**(이력서 단축 + 분석 후행). 목차 순서도 페이지 순서와 동기화.

## 5. 구현 지침 (다음 세션 — "구현해" 지시 시)

전부 **이동**이며 신규 제작물은 #skills 섹션 래퍼뿐이다.

1. **#analysis 섹션 블록 이동**: `<section id="analysis">…</section>` 통째로 잘라 `#unreal`의 `</section>` 바로 뒤에 붙인다.
   (projects를 당기는 게 아니라 analysis를 내리는 편집 — 결과는 동일, diff 최소)
2. **이력서 스킬 패널 분해**: `.rz-skills` 패널에서 `협업`~`SNS` rgroup+`.skills` 6쌍을 잘라내고,
   패널은 `Game Engine — 결과물로 검증` 슬림 행으로 유지(배지 3종). `.skills--engine` 데스크톱 3열 규칙 추가 권장(2+1 낙오 해소).
3. **#skills 섹션 신설** (#analysis 뒤): tag `TOOLBOX` / h2 `기술 · 도구` /
   lede "숙련도는 자기평가입니다 — 실제 증명은 위의 결과물들입니다." / 잘라낸 rgroup+skills 6쌍 그대로.
   CSS는 기존 재사용: `#skills .skills{repeat(2,1fr)}` + `@640 1열` (기존 `.rz-skills .skills` 규칙과 동거 무해, 정리 권장).
   **`.skill--max` 규칙 순서(§2-13 소스 순서 승부) 불변 확인.**
4. **목차 재배열**: `이력서 · 프로젝트 기획서 · 역기획서 · 제안서 · 언리얼 레벨 · 분석 · 리뷰 · 추가 링크` (7개 유지, #skills 미포함).
5. **카피 1건(선택)**: #analysis lede "게임을 뜯어보는 일은 문서 밖에서도 이어집니다"는 그대로도 성립. 강화하려면 "위 기획서 5편 밖에서도".
6. **검증**: 앵커 6종 방향·착지(게임 리뷰 보러가기↓ / 제작 기록↓ / 직접 세운 기록↓ / 리뷰↑×2 / 엔진 배지 #unreal), 넘버링 01–05 불변,
   Lv.10 노랑·마커 잔존, 캐러셀 JS 무손상, 375px 가로 스크롤 0, 콘솔 0, no-JS 링크 생존.
