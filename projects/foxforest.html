Project Overview

01프로젝트 한 눈에 보기

공포·레이싱·퍼즐을 하나의 반복 루프로 통합한 3D 창작 게임입니다. 저는 이 프로젝트의 총괄 기획을 맡아 시스템·콘텐츠·내러티브 설계부터 더미 레벨 제작, 연출, QA까지 담당했습니다.

02시스템 설계 — 규칙 · 상태전이 · 예외 처리

공포·레이싱·퍼즐이 따로 놀 위험을 하나의 코어 루프로 묶고, 모든 사망 조건을 단일 타이머 규칙으로 통일해 개발 공수를 줄였습니다. 조건·출력·예외를 분리한 규칙표로 2인+외주 팀의 해석 차이를 최소화했습니다.

차량 탑승

핵심 규칙 HR01–HR05

HR01

음기 노출 판정

양기 없는 영역 NOTSafeZone에 30초 노출되면 사망. 모든 '죽음' 조건을 하나의 타이머 시스템으로 통일해 재사용.

HR02

양기 안전지대 판정

자동차 내부는 항상 안전지대. 탑승 시 ExposureTimer 정지·리셋.

HR03

등대 접근·해금

자동차 접근으로만 문이 열리고 자동 세이브. 빛 퍼즐 클리어 전까지 다음 Phase 이동 불가.

HR04

보스 AI 감지·추격

SightRange/SoundRange 감지 시 Chase 시작, KillRange 접촉 시 점프스케어 후 사망.

HR05

빛 퍼즐 클리어

빛 경로가 Receiver에 도달 시 클리어 → 시네머신 연출 → 길목 Collider 해제 → 데스 타이머 시작.

상태전이 · 인터럽트 우선순위

PhaseState : 추격·주행 → 주차 판정 → 등대 진입 → PuzzleActive → LighthouseOn → ExposureTimer → VehicleBoarded → NextPhase

Interrupt Priority : Boss_KillRange > ExposureTimer > DeadZone — 동시 발생 시 우선순위로 판정

03이벤트 테이블 — 개발의 핵심 설계도

반복 규칙을 태그로 분리한 설계

같은 규칙을 66개 이벤트마다 반복 서술하지 않기 위해, 공통 동작을 10개 시스템 태그로 분리한 「공통 시스템 규칙표」를 별도로 두었습니다. 이벤트표의 '연결 태그'가 이 규칙표를 참조하는 구조로, 중복 제거 · 유지보수성 · 팀 커뮤니케이션을 동시에 해결했습니다.

Level Design · 서사 전달

05레벨 디자인 & 플레이 속 서사 전달

어두운 숲의 공포는 유지하되 길을 잃지 않게 — 빛·등대·등명기 방향·나선형 계단으로 동선을 설계했습니다. 서사는 장문 설명 대신 플레이 중 발견하고 회수하는 오브젝트로 분산했습니다.

나선형 계단을 택한 3가지 이유

01

유저 직관성

경로를 위/아래 두 축으로 제한해 혼란 감소, 선형 경로 생성

02

개발 편의성

빛 반사 횟수를 줄여 반사체 경우의 수·밸런싱 시간 감소

03

난이도 조정

반사체 경우의 수를 낮춰 퍼즐 난이도 조절이 용이

정보 순서 설계 — 오해에서 진실로

핵심은 "오브젝트를 많이 넣었다"가 아니라, 플레이어가 정보를 어떤 순서로 알게 할지 기준을 세우고 배치했다는 점입니다. 1번 등대는 구미호를 악역으로 오해하게 만들고, 2번 등대에서 진실을 회수하도록 서사 자산의 배치 순서를 설계했습니다.
