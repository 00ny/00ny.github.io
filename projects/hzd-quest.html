TEARDOWN 00 / OVERVIEW

무엇을, 어떻게 정리했나

플레이어가 무엇을 하면(행동) → 게임이 무엇을 확인하고(조건) → 내부에서 무엇이 바뀌고(상태) → 화면에 어떤 반응이 돌아오는지(출력). 이 네 단계를 기준으로 퀘스트 하나를 처음부터 끝까지 뜯어봤습니다.

왜 '가마솥 SIGMA'를 골랐나 INFERRED · 분석자 판단

가마솥은 문이 닫히는 독립 공간이라 날씨·시간·랜덤 이벤트 같은 오픈월드의 변수가 끼어들지 않습니다. 기획자가 배치한 레벨·동선·트리거가 플레이 경험을 그대로 결정하는 곳이라, 이야기와 시스템이 어떻게 맞물리는지 가장 깨끗하게 확인할 수 있는 대상이라고 판단했습니다.

퀘스트 한눈에 보기 OBSERVED · 플레이 확인

이 페이지를 읽는 3개의 라벨

OBSERVED 플레이하면 누구나 확인되는 사실 · INFERRED 관찰을 근거로 제가 정리·해석한 것 · PROPOSED 제가 제안하는 개선안. 세 가지를 섞지 않는 것이 이 문서의 원칙입니다.

TEARDOWN 01 / DIGEST

핵심 요약본 전문 (16p)

"24페이지 원본을 다 읽기엔 부담스럽다"는 독자를 위해 따로 만든 요약 기획서입니다. 바쁘시다면 이 16장만 봐도 전체 분석의 뼈대가 잡히도록 구성했습니다. 클릭하면 확대되고, 화살표·키보드·스와이프로 넘길 수 있습니다.

TEARDOWN 02 / CORE IDEA

이야기가 목표·동선·전투·보상으로 이어지는 구조

'출입이 금지된 기계 제조시설을 탐험하고 정복한다'는 한 줄의 이야기가, 게임 안에서는 4개의 축으로 나뉘어 각각 목표 데이터·지형 배치·전투 설계·보상 설계로 내려갑니다. 이 연결을 확인하는 것이 역기획서의 중심 내용입니다.

이야기 → 목표

'제조시설의 탐험과 정복'이 화면 상단 목표에서 4개의 스텝으로 바뀝니다: 입구 찾기 → 탐색하기 → 코어로 이동하기 → 코어 강제 전환하기.

QS_SIGMA_01 → 02 → 03 → 04 → 05 (순차 완료 체인)
이야기 → 동선

침입을 거부하는 공장의 폐쇄성이, 되돌아갈 수 없는 절벽 강하 지형과 빠른 이동 잠금으로 구현됩니다. 앞으로만 나아가게 만드는 배치입니다.

Player_Control=False · Fast_Travel_Lock=True
이야기 → 전투

인공 시설이라 풀숲 같은 자연 엄폐물이 없는 대신, 환풍구 증기를 이용한 은신 전투가 성립합니다. 보스룸에서는 보호막에 싸인 보스가 기둥(파일런) 3개를 조작해야 깨어납니다.

Player_Visibility=0.0 · Boss_AI_Disable → 전투 시작
이야기 → 보상

코어를 정복한 성취가 일회성 아이템이 아니라, 필드의 기계들을 아군으로 만들 수 있는 'SIGMA 오버라이드 권한'으로 영구히 남습니다.

TEARDOWN 03 / DATA

데이터로 정리한 퀘스트

TEARDOWN 04 / INTENT

설계 의도 해석 — 왜 이렇게 만들었을까

여기서부터는 관찰한 것을 바탕으로 한 저의 해석입니다. 사실이 아니라 추론임을 라벨로 구분합니다.

오버라이드 하나로 모든 것이 열린다 INFERRED

'오버라이드'는 원래 기계를 아군으로 만드는 전투 기술입니다. 그런데 이 던전에서는 같은 조작으로 문을 열고, 다리를 만들고, 보스룸 보호막을 해제합니다. 하나의 버튼이 전투·이동·퍼즐·이야기 진행을 전부 담당하도록 묶어, 퀘스트와 핵심 시스템이 자연스럽게 붙어 있게 만든 설계로 보입니다.

되돌아갈 수 없는 절벽의 두 가지 역할 INFERRED

진입로 끝의 수직 하강은 "이제 앞으로만 가야 한다"는 긴장을 만드는 동시에, 넓은 바깥 필드를 메모리에서 내리고 던전 내부를 불러오는 로딩 화면 없는 전환 통로 역할을 겸한다고 추측합니다. 낙하 직후에 자동 저장을 일부러 갱신하지 않는 것도, 화살이 바닥난 채 갇히는 플레이어가 밖에서 재정비할 수 있게 남겨둔 안전장치로 읽힙니다.

보스전의 시작 버튼을 플레이어에게 준다 INFERRED

마지막 코어룸의 보스는 보호막 안에서 잠든 상태로 기다립니다. 보호막이 유지되는 동안 플레이어는 함정을 깔고, 약점을 살피고, 무기를 정비하며 자기가 준비됐을 때 기둥을 조작해 전투를 시작합니다. 시작 타이밍을 시스템이 아니라 플레이어가 쥐게 한 것이 이 방의 핵심이라고 봤습니다. 주변의 소형 기계와 보급 상자는 탄약이 바닥나도 싸움을 이어갈 수 있게 하는 최소한의 안전망입니다.

TEARDOWN 05 / PROPOSAL

개선 제안 — 진입 경로에 따라 말이 어긋나는 문제 PROPOSED

분석에서 멈추지 않고, 원작에서 발견한 문제 1건과 추가 개발 비용이 거의 없는 해결안을 제안합니다.

발견한 문제

오픈월드에서는 NPC '드랄'을 만나지 않고도 돌아다니다 가마솥 입구에 먼저 도착할 수 있습니다. 그런데 '가마솥'이라는 이름을 들어본 적이 없는 상태에서도 알로이가 "가마솥이군. 들어갈 길이 있을 거야."라고 말합니다. 모르는 이름을 아는 것처럼 말하는 순간입니다.

근거 OBSERVED · 세계관 — 알로이는 부족에서 추방되어 야생에서 자랐습니다. 고대 유적을 금기시하는 환경에서 자란 그녀가 사냥꾼들 사이의 소문인 '가마솥'이라는 이름을 미리 알고 있었을 가능성은 없다고 봐야 합니다.

해결안 — 이미 있는 대사 출력 구조 안에서, 드랄과 대화를 마쳤는지(QUEST_STATUS)를 확인해 상황에 맞는 대사를 내보냅니다. 새로 필요한 것은 대사 한 줄뿐입니다.

IF — 드랄에게 먼저 들은 경우
STR_SIGMA_01A · 기존
"드랄이 말한 가마솥이 여기군. 들어갈 길이 있을 거야."
ELSE — 우연히 먼저 발견한 경우
STR_SIGMA_01B · 신규 작성
"이 기괴한 고대 유적은 대체 뭐지? 들어갈 길을 찾아 보자."

결과 — 어떤 경로로 도착하든 알로이가 아는 만큼만 말하게 되어, 자유로운 탐험과 이야기의 앞뒤가 맞게 됩니다.

TEARDOWN 06 / METHOD

사실 · 해석 · 제안의 경계

원작을 만든 사람이 아닌 제가 쓴 문서인 만큼, 어디까지가 확인된 사실이고 어디부터가 저의 해석과 제안인지 분리해 둡니다.

OBSERVED 플레이하면 확인되는 사실
  • 퀘스트명 · 권장 레벨 8 · 시작 NPC 드랄
  • 12단계 진행 흐름과 화면 반응
  • 핵심 장치: 오버라이드 게이트 · 레펠 강하 · 수송 레일 · 증기 은신 · 기둥 3개 보호막 해제
  • 빠른 이동 잠금 화면 · 보상(4,000XP · 1SP · SIGMA 권한)
  • 알로이의 실제 대사
INFERRED 관찰을 근거로 한 정리·해석
  • 퀘스트/트리거/전투/대사 표 전체와 이름 체계 — 제가 기획서 형식으로 다시 쓴 것
  • 절벽 강하 = 숨은 로딩 통로 · 저장 미갱신 = 안전장치
  • 코어룸 = 시작 버튼을 플레이어에게 준 공간
  • 전투 배치 이유 · 기계 수량/레벨 일부
PROPOSED 분석자의 제안
  • 진입 경로에 따라 대사가 어긋나는 문제 발견
  • 조건 분기 1개 + 신규 대사 1줄의 해결안
  • 추가 개발 비용이 거의 없는 설계

어떻게 관찰했나

특별한 도구 없이, 같은 구간을 조건을 바꿔 여러 번 플레이하는 방식으로 확인했습니다. 목표 문구가 바뀌는 순간의 위치를 앞뒤로 오가며 감지선의 경계를 좁혔고, 드랄과 대화한 저장 파일과 대화하지 않은 저장 파일을 각각 진행해 대사가 갈리는 것을 비교했습니다. 일부러 죽어 부활 위치를 기록했고, 경험치·레벨 같은 수치는 화면 표기를 그대로 옮겼습니다.

배운 것과 한계

이야기와 시스템의 연결을 표로 정리해 보면, 재미와 개연성을 따로가 아니라 한 번에 점검할 수 있다는 것을 배웠습니다. 한계도 분명합니다. 표의 구조와 이름은 제가 붙인 것이라 실제 내부 구현과 다를 수 있고, 기계 수량 일부는 추정입니다. 이 문서의 목표는 '정답 맞히기'가 아니라 '관찰한 것을 개발팀이 쓸 수 있는 형식으로 옮기는 것'입니다.
