Overview

어떤 게임인가 — 1인 2역 보스전

조작

메르세데스(WASD)와 루시드(방향키)를 한 사람이 동시에 조작합니다. 조작 부하 자체가 난이도이자 재미가 되도록 설계했습니다.

전투

모든 보스 패턴이 경고 → 발동 → 판정을 강제하는 ‘보고 피하는’ 전투. 회피 동선과 딜 동선을 백어택 ×2.0 하나로 합쳤습니다.

루프

백화점 허브 → 퀘스트 2라인 → 보스전 해금 → 처치·보상으로 메소 50만 → 드레스 구매 → 엔딩 시네마틱.

Combat Design

“보고 피하는” 보스전 — 패턴을 규칙 언어로 설계

모든 패턴은 경고(텔레그래프) → 발동 → 판정 순서를 강제하고, 각 패턴에 플레이어의 특정 행동을 강제하는 역할을 부여했습니다.

패턴규칙강제하는 플레이기획서의 의도 메모
드라이기5초전방 반원 경고 후 스윕 · 시전 중 방향 고정보스 뒤로 도는 이동 → 백어택 기회 창출“미용실 원장의 정체성 부각”
메스 던지기15초가장 먼 대상에게 투척 → 그 자리로 순간이동 + 슬로우원거리 캠핑 응징 — 두 캐릭터를 모두 관리하게 만듦“기동성 향상 — 메인 스킬” (연계기)
마취주사20초전장 3×3 중 3칸 경고 낙하 · 1칸은 반드시 파티원 위치안전지대 암기 무효화 · 위치 리셋 강제“스킬 적중률 향상”
헤어롤 소환40초4기가 경로 예고 후 돌진 · HP가 있어 요격 가능“피할까, 부술까” 선택지 제공소환술로 전장 오브젝트 추가

패턴 간 공유 시전 간격 5초(기획서 비헤이비어 트리의 ‘공통 스킬 블락’)로 전투 리듬을 통제. 은색/금색 가위와 탈진 게이지(14칸)까지 설계했으나 MVP 스코프에서 의도적으로 컷했습니다.

백어택 — QA로 수치를 바꾼 사례

보스 후방 타격 시 데미지 배수 + CRITICAL 표기. 구현 방식도 기획서에 명세했습니다 — 보스 콜라이더를 4분할(x4)해 후방 판정이 가능하게, “백어택이 가능해, 진형 전술의 이점을 극대화”(기획서 원문). 회피 동선(패턴 피하기)과 딜 동선(뒤 잡기)을 하나로 합치는 핵심 규칙입니다.

×1.5→×2.0

기획서 초기값 1.5배 → QA에서 “뒤를 잡는 리스크 대비 보상이 약하다”고 판단해 2.0배로 상향. 수치 결정과 상향 판단 모두 제가 내렸습니다.

1인 2역 조작 UX

  • 입력 분리 — WASD는 메르세데스, 방향키는 루시드. 전투 중 방향키를 루시드 전용으로 전환하는 규칙을 정의했습니다.
  • Space 한 키로 두 캐릭터 동시 점프 회피 — 가장 짜릿한 순간을 한 키에 압축했습니다.
  • “두 캐릭터가 한 화면을 벗어나지 않는다”는 카메라 유지 규칙을 기획했습니다 (클램프 수치 확정은 개발).

개발자와 소통한 흔적 — 명세가 질문에 답한다

기본공격 하나에도 “플레이어의 인지 비용을 최대한 낮춘다”는 의도 아래 반자동 타게팅(발사 시점 추적, 일직선 투사체, 이후엔 피할 수 있음) 규칙을 그림과 함께 명세했고, 개발자의 질문(“투사체로 자동 타겟팅인가요, 방향키로 조절하나요?”)에 명세서로 답하는 방식으로 협업했습니다.

Balancing

목표 TTK 3:30 — 지표로 돌린 밸런싱 루프

1. 목표 설정보스전 1회 체감 길이 목표를 TTK(처치 시간) 초견 3:30으로 정의
2. 실측QA 플레이로 실제 처치 시간 측정 → 이론값과 괴리 발견
3. 원인 분석요격 가능 소환물(헤어롤)이 화력을 흡수 → 보스 실효 HP가 이론치보다 높음을 규명
4. 데이터 조정메르세데스 딜 사이클 상향(공격 주기 0.77초 등) + 백어택 2.0배로 초견 목표에 수렴 — 숙련 시 절반까지 단축되며, 이 분산이 백어택 숙련의 보상입니다

수치를 한 번 정하고 끝내지 않고, 목표 지표 → 이론 산출 → 실측 → 괴리 원인 → 데이터 수정의 루프를 돌렸습니다. 기획서에서 궁극기 쿨타임 1사이클을 기준으로 한 DPS 산출 모델(레벨·강화 스케일링 포함)을 직접 계산했습니다. 핵심은 값 자체가 아니라, 소환물이 있는 보스전의 실제 난이도는 스탯표가 아니라 플레이어 화력의 분산까지 계산해야 나온다는 발견입니다.

Quest & Narrative

8종 퀘스트(메인 조사 4 + 서브 복수 3 + 메인스토리 1), 2개 라인, 하나의 결말

전투가 ‘어떻게’라면 내러티브는 ‘왜’입니다 — 게임 전체가 루시드가 스승의 꿈에 들어가 보내는 마지막 데이트이고, 겉은 밝은 쇼핑 데이트지만 프레임은 이별입니다. 이 기승전결 액자 구조를 기획서에 명세하고, 아래 퀘스트 2라인으로 구현했습니다.

메인 — JM 조사 1~4 수사 라인

증거품(킹슬라임 꼬리 → 악어이빨 → 피 묻은 메스)을 모아 백화점 소동의 범인을 추적합니다. 완주 보상이 최종 포탈 개방 + 대형 메소로 이어져 엔딩 조건을 지원합니다.

서브 — 수선화의 복수 1~3 추리 라인

억울하게 죽은 ‘정체불명의 그녀’의 복수 대행. 성형 전/후 사진 2장을 단서로 용의자 3명 중 진짜 찾기 — 퀘스트 완료 표시(UI)를 의도적으로 숨겨 정답을 감췄습니다. 완료 시 보스전 입장 권한(현상수배지)이 열립니다.

라인 교차 설계 — 서브 라인이 전투 콘텐츠(보스)를 열고, 메인 라인이 경제(메소)를 엽니다. 두 라인을 모두 돌아야 엔딩 조건(메소 50만·드레스 구매)에 도달하며, 보상은 엔딩 기준으로 역산 배분했습니다.

세계가 반응하는 후일담 — 보스 처치 후 ‘강화안내원 → 리나’ 교체, 복수 완료 후 그녀는 성불해 맵에서 사라집니다. NPC 배치도 제작과 실제 배치 작업까지 직접 수행했습니다.

Process

5인 팀을 굴린 관리 체계

기능 백로그 sys_01~101

기능 102건을 대분류/상세/담당자/개발상태로 관리. 우선순위를 P0(게임이 돌아감)~P4로 등급화하고, 각 행을 기획서 페이지·기획상태(정의완료/미정의)와 연결해 개발자가 문서를 뒤지지 않고 착수하게 했습니다.

스코프 2벌 운영 (풀 ↔ MVP)

퀘스트 테이블을 ‘풀 기획’과 ‘MVP’ 두 벌로 운영하고, 시트에 “[MVP]로 이 항목 삭제/임시변경” 같은 컷 결정을 항목 단위로 기록. 컷한 것(속성 상성 8×8·가위 패턴 2종·탈진 게이지·강화·대쉬·호감도 분기·뷰 전환)과 지킬 것(보스전 코어·퀘스트 2라인·엔딩)을 처음부터 분리 — 3주 완주의 핵심이었습니다.

아트 파이프라인 시트

리소스별 RUID/담당자/진행 상태/우선순위/마감을 추적하고, 컷별 필요 원화 지시를 대사 원고에 병기해 아트 요청을 문서화했습니다.

AI 협업 문서화

개발 종반, AI(Claude)에게 코드베이스 전수 분석을 지시·검수해 구현 기준 아키텍처 문서(CLAUDE.md)와 기획자용 구조 문서를 확정 — 기획-구현 정합성을 문서로 고정하는 파이프라인을 기획자가 주도했습니다.

Role Boundary

역할 경계 — 정직 버전

내가 설계·작성

  • 보스 패턴 행동 규칙 전부 (기획 의도 포함)
  • 백어택 규칙 (콜라이더 4분할 · 1.5 → 2.0 상향)
  • 퀘스트 8종 데이터·대사 전량 (250행+)
  • 다이얼로그 4표 분리 스키마
  • 경제(메소 50만)·보상 역산 배분
  • NPC 배치·동선 / 포탈·회복템·경계 규칙
  • 관리 시트(sys 101)·DPS 산출·밸런싱 루프(TTK 3:30)

협업으로 확정

  • 드라이기 범위 — 그림 가이드 → 각도(180°)는 테스트 협의
  • 이동속도 — ‘빠름/느림’ 가이드 → 수치는 개발 확정
  • 분리거리 클램프 — 화면 유지 규칙 기획 → 수치는 개발 확정

개발 파트 단독

  • mLua 구현 전체 (156파일)
  • 플랫폼 제약 우회 구조 (QuestBridge)
  • 대사 마커 체계·선택지 포맷 고안

면접에서 첫째 열은 어디까지든 설명할 수 있고, 셋째 열은 제 것이 아니라고 즉시 말합니다.
